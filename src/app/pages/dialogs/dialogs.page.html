Description

It can be triggered by any element with the "modal-trigger" slot.
By default it has a close button and action buttons (close and ok), but you can remove them by setting closeButton and actionButtons to false.

Attributes list:

  • style (object, default: {{ '{' }}{{ '}' }}) - Custom CSS styles as key-value pairs
  • class (string, default: '') - CSS class names to apply to the modal
  • id (string, default: '') - HTML id attribute for the modal dialog element
  • width (string, default: '') - Custom width for the modal (e.g., '500px', '50%')
  • height (string, default: '') - Custom height for the modal (e.g., '400px', '80vh')
  • closeButton (boolean, default: true) - When true, displays a close button in the top-right corner
  • actionButtons (boolean, default: true) - When true, displays action buttons (Close and Ok) at the bottom
  • onConfirm (function, default: () => {{ '{' }}{{ '}' }}) - Callback function executed when the Ok button is clicked
  • onClose (function, default: () => {{ '{' }}{{ '}' }}) - Callback function executed when the modal is closed

Content Slots:

  • slot="modal-trigger" - Element(s) that will trigger the modal to open when clicked (e.g., button, link, icon)
  • slot="modal-content" - Content to be displayed inside the modal body (scrollable area)

Public Methods:

  • openModal() - Opens the modal programmatically
  • closeModal() - Closes the modal programmatically
  • isOpen() - Returns boolean indicating if the modal is currently open
  • confirm() - Triggers the onConfirm callback and closes the modal


Toast

Injectable service for toast notifications



Description

ToastService is an injectable service that displays temporary notification messages.
Toasts appear from the bottom-right corner with smooth animations and auto-dismiss after a specified duration.
Multiple toasts stack vertically with 16px spacing between them.
Toasts have max-width: 260px, min-height: 150px and expand based on content.
Click any toast to dismiss it early.

Methods:

  • toast(options: ToastOptions) - Displays a success/info toast with default styling (border: var(--nc-toast-border), background: var(--nc-toast-bg))
  • error(options: ToastOptions) - Displays an error toast with error styling (border: var(--nc-error-toast-border), background: var(--nc-error-toast-bg))

ToastOptions Interface:

  • text (string, required) - The message to display in the toast
  • duration (number, optional, default: 2500) - Duration in milliseconds before the toast auto-dismisses

Features:

  • ✓ Fixed position at bottom-right corner (right: 20px, bottom: 20px)
  • ✓ Smooth slide-in/slide-out animations (300ms transition)
  • ✓ Multiple toasts stack vertically with automatic repositioning
  • ✓ Click to dismiss any toast early
  • ✓ Dynamic DOM creation - no template elements needed
  • ✓ Max width: 260px, Min height: 150px (expands with content)
  • ✓ Injectable service - use anywhere in your application
