CRUD List with Custom Templates

Render custom HTML for each item using templates



Description

For advanced customization, you can pass a TemplateRef to individual items to render custom HTML instead of plain text.

Steps:

  • 0. Read the HTML & TS below if the following lines sound too abstract:
  • 1. Create <ng-template> elements in your HTML with template reference variables (e.g., #myTemplate)
  • 2. Add @ViewChild('myTemplate') myTemplate!: TemplateRef<any>; in your TypeScript component
  • 3. Implement AfterViewInit lifecycle hook to assign templates to items after view initialization
  • 4. Assign the template to the template property of specific items in ngAfterViewInit()
  • 5. Call this.cdr.detectChanges() to trigger change detection (inject ChangeDetectorRef in constructor)
  • 6. Use let-item and let-i="index" in your template to access item data and index


Result

HTML

TypeScript



Attributes list:

  • items (CrudListItem[], required) - Array of items to display in the list
  • title (string, default: 'Item List') - Title displayed in the list header
  • striped (boolean, default: true) - Alternating row colors for better readability
  • noItemsText (string, default: 'No items found') - Message shown when list is empty
  • createButton (boolean, default: true) - Show/hide the create (plus) button
  • updateButton (boolean, default: true) - Show/hide update (edit) buttons
  • deleteButton (boolean, default: true) - Show/hide delete buttons
  • onCreate (function) - Callback when create button is clicked
  • onClick (function) - Callback when item text is clicked
  • onUpdate (function) - Callback when update button is clicked
  • onDelete (function) - Callback when delete button is clicked
  • class (string, default: '') - CSS classes to apply to the component
  • style (object, default: {{ '{' }}{{ '}' }}) - Inline styles to apply
  • id (string, default: '') - HTML id attribute

CrudListItem Interface:

  • text (string, required) - Text to display for the item
  • template (TemplateRef<any>, optional) - Custom template to render instead of text
  • data (any, optional) - Additional data to store with the item
  • onClick (function, optional) - Item-specific click handler
