Basic Table (Non-Virtualized)

A full-featured table for smaller datasets

Description

The Table component (<nc-table>) provides the same features and appearance as the Virtualized Table, but renders all data rows at once without virtualization. Use this component when you have smaller datasets (typically under 100-200 rows) where virtualization isn't necessary.

When to Use

  • Smaller Datasets: Best for datasets with fewer than 200 rows
  • Simpler Implementation: All rows rendered at once, no virtual scrolling complexity
  • Full Compatibility: Same props, features, and styling as the virtualized table
  • Easier Debugging: All rows in DOM, easier to inspect with browser dev tools

Result



Features

The <nc-table> component includes all the same features as <nc-virtualized-table>:

  • Filtering: Advanced filtering with multiple conditions
  • Sorting: Click column headers to sort
  • Column Visibility: Show/hide columns dynamically
  • Zoom: Adjust table content size
  • Filter Presets: Save and apply common filters
  • Drag & Drop: Reorder columns
  • Striped Rows: Alternating row colors
  • Hover Effects: Highlight rows on hover

Virtualized vs Regular Table

Use <nc-table> when you have smaller datasets and want simpler rendering.
Use <nc-virtualized-table> when you have large datasets (200+ rows) and need optimal performance through virtual scrolling.

Virtualized Table - Column Widths & Row Heights

Customize column widths and understand adaptive row heights

Column Width Behavior

Default Behavior: By default, the virtualized table automatically distributes its width equally among all columns. This ensures a balanced layout without any configuration.

Custom Column Widths: If you want to set custom column widths, you must define the width for ALL columns using the columnsConfig prop. You cannot mix custom and auto-width columns. Define widths in any CSS unit (px, %, rem, etc.).

Row Height Behavior

Row heights are automatically adaptive and adjust dynamically based on the content within each cell. If a cell contains text that wraps to multiple lines, the entire row expands to accommodate the tallest cell. This ensures all content is visible without truncation.

Table Height Customization

The default maximum table height is 600px. You can customize this by overriding the CSS or using inline styles to adjust the max-height property of the table body. See examples below.

Result



Key Features

  • Equal Distribution: By default, columns share width equally across the table
  • Custom Widths: Set specific widths by defining ALL columns in columnsConfig
  • Adaptive Heights: Rows automatically expand to fit multi-line content
  • Configurable Table Height: Override max-height via CSS or inline styles
  • Text Wrapping: Long text wraps naturally within cells without truncation

Virtualized Table - Columns Config

Reorder and rename columns programmatically or interactively

Column Reordering

You can control the display order of columns by specifying them in your desired sequence within columnsConfig. The order in which you define columns in the configuration array determines how they appear in the table, regardless of the order of fields in your data objects.

Column Renaming

Use the displayValue property to provide user-friendly column headers that differ from your data field names. For example, you can display "User ID" as the header while the data field is named id, or rename email to "Email Address" for better readability.

Interactive Features

Drag & Drop Reordering: Users can click and drag column headers to reposition them dynamically. The new order persists throughout the user's session, allowing for personalized table layouts.

Show/Hide Columns: When controls is enabled, users can click the column visibility button (list icon) in the control bar to toggle individual columns on or off. Hidden columns are completely removed from the display and filtering operations.

Result

Notice how the columns are reordered (Name first, ID last) and renamed with more descriptive labels. Try dragging column headers to reorder them or use the column visibility control!



Key Features

  • Programmatic Reordering: Define column order via columnsConfig array sequence
  • Column Renaming: Use displayValue to show user-friendly headers
  • Data Independence: Column order is independent of data object field order
  • Interactive Drag & Drop: Users can reorder columns by dragging headers
  • Column Visibility Control: Show/hide columns via the controls bar button
  • Persistent Changes: User modifications persist during the session
